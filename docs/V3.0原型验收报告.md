## 四、遗留问题 / 建议

1. **双端缺口**：画布 2 仅手机端 6 页，V3.0 的平板适配页（如 `平板-账号管理页`、`平板-数据导出页`、`平板-应用题专项页`）未绘制，建议补绘以保持与 V1/V2 一致的"手机+平板"双端策略。
2. **跨画布导航**：画布 1 的「专项进阶」入口需能跳转至画布 2 的「应用题专项」，确认跳转链路与 L5 解锁条件（在先 V2 难度页显示"L5 已解锁"入口）。
3. **数据模型**：多账号切换要求进度/积分/错题/报告按 child_id 隔离，开发侧需确认数据层已支持。
4. **可选项**：错题精准讲解页的"图形化演示"是否复用「可视化教学页」的动画组件，建议统一以免重复开发。
